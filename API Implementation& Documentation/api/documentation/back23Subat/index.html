BloodHub

This is our high-quality API. This documentation is only for administrative purpose only. API won't be running without proper secret code taken by the administartion unit.

Android and the iOS apps will be using this API channel to communicate with servers of the BloodHub.

API

Below are the fallback services that are used by the BloodHub Android and iOS apps.

List Users

Lists all users in the system. You need to have a proper secretCode to use the API Service.

Property Description
secretCode ( required ) the code taken from the administration
available ( optional ) takes 1 or 0 values. If not specified default 1 .
GET
/api/v1/getUsers?available={available}&secretCode={secret_key}

$ curl http://cs491-2.mustafaculban.net/api/v1/getUsers?available={available}&secretCode={secret_key}

Create a User

Creates a newuser.

Property Description
username ( required ) username of the user.
firstname ( required ) name of the user.
surname ( required ) surname of the user.
password ( required ) password of the user. Need to be given as raw input, non-hashed.
email ( required ) email of the user.
identityNum ( required ) identitiy number of the user, both foreign and turkish citizen number are valid.
bloodType ( required ) blood type of the user, should be given as %2B for + type of blood. check encoding reference
birthdate ( required ) birthdate of the user, should be given as this pattern: ddmmYYYY .
address ( required ) address of the user
telephone ( required ) telephone of the user
secretCode ( required ) the code taken from the administration
GET
/api/v1/register/{request_body}

Request body

{
  "username": "xxx",
  "firstname": "xxx",
  "surname": "xxx",
  "password": "xxx",
  "email": "xxx",
  "identityNum": "xxx",
  "bloodType": "xxx",
  "birthdate": "xxx",
  "address": "xxx",
  "telephone": "xxx",
  "secretCode": "xxx"
}

Example request

curl -X GET http://cs491-2.mustafaculban.net/api/v1/register/?username={username}&firstname={firtsname}&surname={surname}&password={password}&email={email}&identityNum={identityNum}&bloodType={bloodType}&birthdate={birthdate}&address={address}&telephone={telephone}&secretCode={secret_key}

Send Blood Request

Sending the blood request via API. Whole Request body should be sent to request.

Property Description
name_surname ( required ) the full name of the patient.
notificationType ( required ) one of these 3 type of notification. { mail , sms , push }
bloodType ( required ) blood type of the user, should be given as %2B for + type of blood. check encoding reference
location ( required ) the destination location of the hospital
hospitalName ( required ) the full hospital name
secretCode ( required ) the code taken from the administration.
GET
/api/v1/sendBloodRequest/{request_body}

Request body

{
  "name_surname": "xxx",
  "notificationType": "xxx",
  "bloodType": "xxx",
  "location": "xxx",
  "hospitalName": "xxx",
  "secretCode": "xxx"
}

Example request

curl -X GET "http://cs491-2.mustafaculban.net/api/v1/sendBloodRequest/?name_surname={name_surname}&notificationType={notification_type}&bloodType={bloodType}&location={location}&hospitalName={hospitalName}&secretCode={secretCode}"

Example response

{
  "owner": "{username}",
  "id": "{id}",
  "name": "foo",
  "description": "bar",
  "created": "{timestamp}",
  "modified": "{timestamp}"
}

Add Event

Adding event with API. These event will be shown on the android and iOS apps.

Property Description
name ( required ) The full name of the event.
city ( required ) City where this event takes place.
content ( required ) Full content of the vent . Content can include HTML codes.
latitude ( required ) Latitude of the event where it happens. Type should be Double
longitude ( required ) Longitude of the event where it happens. Type should be Double
startDate ( required ) Time when this event starts
endDate ( required ) Time when this event ends.
secretCode ( required ) the code taken from the administration.
GET
/api/v1/addEvent/{request_body}

Request body

{
  "name": "xxx",
  "city": "xxx",
  "content": "xxx",
  "startDate": "xxx",
  "endDate": "xxx",
  "latitude": 11,
  "longitude": 11
}

Example request

curl -X GET "http://cs491-2.mustafaculban.net/api/v1/addEvent/?name={name}&city={city}&latitude={latitude}&longitude={longitude}&content={content}&startDate={startDate}&endDate={endDate}&secretCode={secretCode}"

Example of erroneous response

{
    "error": "TRUE",
    "error_msg": "Error on saving event into database."
}

Example of succesful response

{
    "error": "FALSE",
    "success": "Event was added succesfully.",

Get Event

Property Description
