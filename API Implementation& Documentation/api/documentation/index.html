BloodHub

This is our high-quality API. This documentation is only for administrative purpose only. API won't be running without proper secret code taken by the administartion unit.

Android and the iOS apps will be using this API channel to communicate with servers of the BloodHub.

Below are the fallback services that are used by the BloodHub Android and iOS apps.

Create a User

Creates a newuser.

Property Description
username ( required ) username of the user.
firstname ( required ) name of the user.
surname ( required ) surname of the user.
password ( required ) password of the user. Need to be given as raw input, non-hashed.
email ( required ) email of the user.
identityNum ( required ) identitiy number of the user, both foreign and turkish citizen number are valid.
bloodType ( required ) blood type of the user, should be given as %2B for + type of blood. check encoding reference
birthdate ( required ) birthdate of the user, should be given as this pattern: ddmmYYYY .
address ( required ) address of the user
telephone ( required ) telephone of the user
secretCode ( required ) the code taken from the administration

/api/v1/register/{request_body}

{
  "username": "xxx",

  "bloodType": "xxx",

Send Blood Request

Sending the blood request via API. Whole Request body should be sent to request.

Property Description
name_surname ( required ) the full name of the patient.
notificationType ( required ) one of these 3 type of notification. { mail , sms , push }
bloodType ( required ) blood type of the user, should be given as %2B for + type of blood. check encoding reference
location ( required ) the destination location of the hospital
hospitalName ( required ) the full hospital name
senderID ( required ) which user created that notification.
secretCode ( required ) the code taken from the administration.
GET
/api/v1/sendBloodRequest/{request_body}

Request body

{
  "name_surname": "xxx",
  "notificationType": "xxx",
  "bloodType": "xxx",
  "location": "xxx",
  "hospitalName": "xxx",
  "secretCode": "xxx"
}

Example request

curl -X GET "http://cs491-2.mustafaculban.net/api/v1/sendBloodRequest/?name_surname={name_surname}&notificationType={notification_type}&bloodType={bloodType}&location={location}&hospitalName={hospitalName}&senderID={senderID}&secretCode={secretCode}"

Example response

{
    "error": false,
    "success": "Message was sent to the nearby devices succesfully."
}

limit ( optional ) limit the fetched data . It is 10 by default.
type ( optional ) type of the received notifications. Must be one of these 3: {'push', 'mail','sms'}

api/v1/getSentNotification/{senderID}/&secretCode={secretCode}&*limit={limit}*&*type={type}*

Approve/Reject EM5 Request

This endpoint is used to approve/reject the request created to add a user by the owner of the Emergency Five(EM5) list. Here the order in the list is important for user to approve/reject the request.

Property Description
ownerID ( required ) ID of the owner of the EM5 list.
uid ( required ) Id of the user who wants to accept/reject the request created by owner.
order ( required ) order of the user where he/she was aded in to the owner's list.
This can be one of the following enumerated list: {'first','second', 'third', 'fourth', 'fifth'}
choice ( required ) choice of the request user selects. This can be one of the following: {'-1', '0', '1'}
'-1' for rejecting the request,
'0' is set as initial value for request by system,
'1' for accepting the request.
GET
api/v1/approveEM5Request/?ownerID=1&uid=3&order=third&choice=-1&secretCode={secretCode}

Example request

curl -X GET "http://cs491-2.mustafaculban.net/api/v1/approveEM5Request/?ownerID=1&uid=2&order=first&choice=1&secretCode=xxxx

Example for successfuly update on the EM5 list for the owner of it.

{
    "error": "FALSE",
    "success": "You have successfully updated EM5 List preferences of yourself on the owner list."
}

Example for error due to trying to update a user in the wrong order.

{
    "error": "TRUE",
    "error_msg": "This user is not in that order. Plase specify the order of the user."
}

Example for error in the Server.

{
    "error": "TRUE",
    "error_msg": "There is a problem while updating EM5 List. Please try again later."
}
